Pré-alocação de Memória em R

Introdução

O que é pré-alocação de memória?

Dado que precisaremos de um tamanho limitado e conhecido de memória para uma execução de um programa, podemos via software “solicitar” esse espaço ao hardware.

Por que é importante?

Gera menor tempo de execução e menor cu$to.

Exemplo sem Pré-alocação

out <- c()
for (i in 1:4) {
  out <- c(out, i)
  print(out)}
[1] 1
[1] 1 2
[1] 1 2 3
[1] 1 2 3 4
out <- c()
for (i in 1:4) {
  out <- c(out, i)
  cat('It', i, '| memory address:', lobstr::obj_addr(out), '\n')}
It 1 | memory address: 0x60697ac04798 
It 2 | memory address: 0x60697ac4e2e8 
It 3 | memory address: 0x60697a7e6078 
It 4 | memory address: 0x60697a7e5f38

Exemplo com Pré-alocação

out <- rep(NA, 4)
for (i in 1:5) {
  out[i] <- i
  cat('It', i, '| memory address:', lobstr::obj_addr(out), '\n')}
It 1 | memory address: 0x606979293e68 
It 2 | memory address: 0x606979293e68 
It 3 | memory address: 0x606979293e68 
It 4 | memory address: 0x606979293e68 
It 5 | memory address: 0x60697aa21928

Obs: Nesse caso alocamos um vetor de 4 espaços e iteramos 5 vezes. Em R, esse objeto tem parte pré-alocada e parte dinâmica.

Tempo

n <- 10000

for_loop_timing <- microbenchmark(

  'Dinâmica' = {
    out <- c()
    for (i in 1:n) { out <- c(out, i) }
  },
  'Pré-alocada' = {
    out <- rep(NA, n)
    for (i in 1:n) { out[i] <- i }
  },
  times = 100
)

Tempo

Funções do R padrão

Funções apply em R (apply, lapply, sapply, vapply…) internamente pré-alocam memória , por isso são geralmente tratadas como “mais eficientes” que usar for ou while.

for_loop_timing <- microbenchmark(

  'Dinâmica' = {
    out <- c()
    for (i in 1:n) { out <- c(out, i) }
  },
  'Pré-alocada' = {
    out <- rep(NA, n)
    for (i in 1:n) { out[i] <- i }
  },
  'Apply' = {
    out <- lapply(1:n, function(x) x)
  },

  times = 100
)
